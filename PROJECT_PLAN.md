## Предварительный план проекта

* **Цель:** создать простую песочницу в ASCII‑стиле, где игрок может взаимодействовать с миром (например, перемещаться, собирать предметы, изменять мир)
* **Подход:** сначала делаем минимальный рабочий прототип (MVP), потом постепенно добавляем механики
* **Итерации:** можно делить проект на небольшие шаги, каждый шаг добавляет что-то новое в мир или в логику игры

## Описание архитектуры проекта

* **Сущности и компоненты:** все объекты мира (игрок, предметы, элементы карты) можно представить как сущности с простыми свойствами, например: позиция, отображаемый символ, цвет
* **Система обновления:** отдельный код отвечает за то, что происходит с этими объектами (движение, взаимодействие)
* **Рендеринг:** отдельный код отвечает только за вывод символов на экран, цвета, фон не смешивать с логикой игры
* **Мир:** двумерная сетка клеток, каждая клетка хранит символ и цвет

## Выбор технологий и инструментов

* **Язык:** C++
* **Библиотека для ASCII:** можно использовать `libtcod` (упрощает рендер символов и цвет), либо писать через стандартный вывод в консоль
* **Сборка:** CMake
* **Контроль версий:** Git + GitHub
* **Файлы данных:** YAML для хранения конфигураций (например, виды объектов, символы, цвета)
* **IDE:** Visual Studio

## Рабочий процесс (workflow)

* Основные ветки:
  * `master` стабильная версия, которую можно запускать
  * `dev` ветка для разработки
* Каждая новая функция делается в отдельной ветке от `dev`, потом сливается через Pull Request после обсуждения и апрува
* Краткие коммиты, по возможности Issues для отслеживания того, что нужно сделать
* README.md объясняет, как собрать и запустить игру
